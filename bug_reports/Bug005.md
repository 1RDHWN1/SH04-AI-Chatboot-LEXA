## Title
Unbounded Chat History Causes Increasing Latency and Memory Growth in Long Sessions

## Severity
🟠 **High**

## Priority
🟠 **P2 — Fix Before Production Deployment**

---

## Description

The `LexaChatbot.history` list accumulates all conversation turns indefinitely. With each message exchange, two new entries are appended (`user` + `assistant`). The complete history is sent with every API request.

**Consequences:**
1. **Increasing API request payload size** → higher token counts → higher latency.
2. **Potential token context window overflow** → API 400 error at very long sessions.
3. **Growing memory usage** → ~138 MB at 100 turns (measured in PT-006).
4. **Increasing API costs** (if on paid tier) proportional to history length.

At 100 conversation turns, the history contains 201 entries (1 system + 100 user + 100 assistant). Estimated token count: ~20,000–30,000 tokens per request.

---

## Steps to Reproduce

1. Initialize `LexaChatbot()`.
2. Call `send_message()` 100+ times.
3. Observe: `len(bot.history)` = 201.
4. Observe increasing response latency per turn.
5. At ~200+ turns, Groq API may return 400 context window exceeded error.

---

## Expected Result

Chat history is bounded. Old messages beyond a configurable limit are pruned or summarized. Latency remains consistent regardless of session length.

---

## Actual Result

History grows without bound. Latency increases linearly with session length. Memory usage grows to 138MB at 100 turns.

---

## Recommendation

**Option A — Sliding Window (Simple):**
```python
MAX_HISTORY_TURNS = 20  # Keep last 20 exchanges (40 messages + system prompt)

def _prune_history(self):
    """Keep system prompt + last MAX_HISTORY_TURNS exchanges."""
    system_prompt = self.history[0]
    conversation = self.history[1:]  # All except system
    if len(conversation) > MAX_HISTORY_TURNS * 2:
        conversation = conversation[-(MAX_HISTORY_TURNS * 2):]
    self.history = [system_prompt] + conversation

def send_message(self, message: str) -> str:
    self._validate_message(message)
    self._prune_history()  # Prune before each request
    self.history.append({"role": "user", "content": message})
    # ... rest of method
```

**Option B — Token-Based Pruning (Advanced):**
Use a tokenizer to estimate token count and prune when approaching 4,096 tokens (or model's context limit).

**Option C — Summarization (Best UX):**
When history exceeds limit, send a summary request to the LLM: "Summarize the conversation so far in 3 sentences," and replace old history with the summary.
