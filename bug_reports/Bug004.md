## Title
Rate Limit Errors (HTTP 429) Not Handled with User-Friendly Message or Retry Logic

## Severity
🟠 **High**

## Priority
🟠 **P2 — Fix in Next Sprint**

---

## Description

When the Groq API returns a `429 Rate Limit Exceeded` error, the application catches it generically and raises a `RuntimeError` with a technical error string. No user-friendly message, cooldown indicator, or retry mechanism is provided.

In the CLI, the error prints as:
```
Terjadi kesalahan: Gagal memproses request ke Groq API: Error code: 429...
```

In Streamlit:
```
st.error("Terjadi kesalahan saat memproses jawaban: Gagal memproses request ke Groq API: ...")
```

Users see a red error banner with a technical Groq API error message and have no guidance on what to do or when to retry.

---

## Steps to Reproduce

1. Configure a Groq API key with a low rate limit (free tier).
2. Send 20+ rapid messages in the CLI or Streamlit.
3. Observe the error output when rate limit is hit.

---

## Expected Result

- A friendly message: `"Sedang ramai, silakan coba lagi dalam beberapa detik."`
- Optionally: automatic retry after 2–5 seconds (with user notification).
- No technical API error strings exposed to the user.

---

## Actual Result

Technical `RuntimeError` string containing `"429"` and Groq internal error text displayed directly to the user.

---

## Recommendation

```python
from groq import RateLimitError

def send_message(self, message: str) -> str:
    self._validate_message(message)
    self.history.append({"role": "user", "content": message})
    
    try:
        chat_completion = self.client.chat.completions.create(
            messages=self.history,
            model=self.model,
        )
        reply = chat_completion.choices[0].message.content
        self.history.append({"role": "assistant", "content": reply})
        return reply
        
    except RateLimitError:
        self.history.pop()
        raise RuntimeError(
            "Sistem sedang ramai. Silakan tunggu beberapa detik dan coba lagi."
        )
    except Exception as e:
        self.history.pop()
        raise RuntimeError(f"Gagal memproses request: {e}")
```
