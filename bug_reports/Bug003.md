## Title
Hardcoded CSS Background Color Breaks Streamlit Dark Mode

## Severity
🟡 **Medium**

## Priority
🟡 **P3 — Fix in Next Release**

---

## Description

The `app.py` file injects a custom CSS block using `st.markdown(..., unsafe_allow_html=True)` that includes:

```css
.main {
    background-color: #f8fafc;
}
```

This hardcoded light-gray background overrides Streamlit's native dark mode theme. When users switch to dark mode via Streamlit's settings, the chat area remains white/light, while the sidebar and other components switch to dark — creating a visually inconsistent and difficult-to-read interface.

---

## Steps to Reproduce

1. Run `streamlit run app.py`.
2. Click the hamburger menu (top right) → Settings.
3. Change theme from "Light" to "Dark".
4. Observe the main content area.

---

## Expected Result

The entire UI adapts to dark mode. Background becomes dark (#0e1117 or similar). Text remains readable with light foreground colors.

---

## Actual Result

Main content area remains white/light gray (`#f8fafc`) while the sidebar and other components switch to dark mode, creating a jarring visual inconsistency. Text color contrast may be insufficient.

---

## Recommendation

**Option A — Remove the hardcoded background and let Streamlit manage theming:**
```python
# Remove this block entirely:
st.markdown("""
    <style>
    .main {
        background-color: #f8fafc;  ← DELETE THIS
    }
    ...
    </style>
""", unsafe_allow_html=True)
```

**Option B — Use a `.streamlit/config.toml` for theming:**
```toml
# .streamlit/config.toml
[theme]
base = "light"
primaryColor = "#ef4444"
backgroundColor = "#f8fafc"
textColor = "#0f172a"
```

This approach applies theming at the Streamlit level and respects the user's theme override.
