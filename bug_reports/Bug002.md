## Title
No Input Validation in `send_message()` and `send_message_stream()` — Empty, Null, and Oversized Inputs Bypass Guards

## Severity
🟠 **High**

## Priority
🟠 **P2 — Fix in Next Sprint**

---

## Description

The `LexaChatbot.send_message()` and `send_message_stream()` methods in `llm.py` contain no validation for the `message` parameter. This means:

1. **Empty strings** (`""`) are forwarded directly to the Groq API.
2. **`None`** values cause a `TypeError` deep in the SDK stack, surfacing a confusing error.
3. **Extremely large inputs** (10,000+ characters) exceed API token limits, causing generic errors with no user-friendly message.
4. **Whitespace-only strings** bypass the CLI guard if called programmatically.

The existing guard in `main.py` (`if not user_input.strip(): continue`) only protects the CLI interface. Direct calls to `llm.py`, or calls via Streamlit, have no equivalent protection at the library level.

---

## Steps to Reproduce

**Case A — Empty string:**
```python
from llm import LexaChatbot
bot = LexaChatbot()
bot.send_message("")  # Sent directly to API
```

**Case B — None value:**
```python
bot.send_message(None)  # TypeError in SDK
```

**Case C — Large input:**
```python
bot.send_message("A" * 10000)  # Token limit exceeded
```

---

## Expected Result

- `send_message("")` → raises `ValueError("Pesan tidak boleh kosong.")`
- `send_message(None)` → raises `TypeError("Pesan harus berupa string.")`
- `send_message("A" * 10000)` → raises `ValueError("Pesan terlalu panjang. Maksimum 4000 karakter.")`

---

## Actual Result

- Empty string: API called; result model-dependent (may error or return prompt to clarify).
- None: `TypeError` surfaces from SDK internals with no helpful message.
- Large input: API returns 400/token limit error wrapped in `RuntimeError`.

---

## Recommendation

Add an input validation block at the top of both methods:

```python
def _validate_message(self, message):
    if message is None:
        raise TypeError("Pesan harus berupa string, bukan None.")
    if not isinstance(message, str):
        raise TypeError(f"Pesan harus berupa string, bukan {type(message).__name__}.")
    if not message.strip():
        raise ValueError("Pesan tidak boleh kosong atau hanya berisi spasi.")
    MAX_CHARS = 4000
    if len(message) > MAX_CHARS:
        raise ValueError(f"Pesan terlalu panjang ({len(message)} karakter). Maksimum {MAX_CHARS} karakter.")

def send_message(self, message: str) -> str:
    self._validate_message(message)
    # ... rest of method
```
